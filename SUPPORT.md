# Support

Use [GitHub Discussions](https://github.com/ap-in-indy/aiignore/discussions) for
usage questions and design discussion after the repository is public. Use
[GitHub Issues](https://github.com/ap-in-indy/aiignore/issues) for reproducible
bugs and the RFC template for normative proposals.

This experimental project has no commercial support commitment or response-time
guarantee. Do not post credentials, private policies, customer data, or
unredacted harness transcripts. Report vulnerabilities through
[SECURITY.md](SECURITY.md).

Only the latest published prerelease and current `main` are supported during
the alpha. Publishing a newer prerelease ends routine support for the prior
one; older artifacts remain available for reproducibility and may receive a
public advisory directing users to upgrade. No alpha receives long-term
support. See [docs/versioning.md](docs/versioning.md) for compatibility and
end-of-life rules.

Support covers reproducible defects in the published specification, reference
implementation, package, vectors, and versioned adapters. It does not include a
promise that a vendor harness mediates an undocumented interface, deployment
consulting, incident response for downstream environments, or compatibility
with unpinned prerelease versions.
